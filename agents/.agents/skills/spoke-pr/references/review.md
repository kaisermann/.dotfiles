## Review A PR

- Read the title and body before diving into the diff.
- If a large PR lacks context, ask for it early.
- Review for:
  - correctness and regressions
  - scope and reviewability
  - testing quality
  - rollout or migration safety
  - cross-repo coordination and merge order
  - design fidelity for UI work
  - missing context that blocks confident review

### Review comments

- Separate blockers from suggestions.
- Block on correctness, important risk, fragility, or not-being-ready-to-merge.
- Do not turn taste or minor nits into blockers.
- Say what you did not review when coverage was partial.
